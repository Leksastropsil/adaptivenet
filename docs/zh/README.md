# Edge Adapter Framework

_(研究与教育项目)_

一个 **基于边缘计算 (Edge Computing)** 的框架，采用 **Core → Adapters** 架构，用于研究 **现代网络系统行为**、边缘编排以及平台互操作性与保护模式（速率限制、反机器人等），在 **教育与技术研究** 的背景下使用。

本项目关注 **系统架构与软件工程**，而非任何特定网站或内容。

## 🧠 概述

现代网络平台使用各种保护机制，例如：

- 机器人检测 & 指纹识别
- 速率限制 & IP 声誉
- 基于 JavaScript 的动态挑战

本框架旨在通过以下方法 **学习并模拟** 这些模式：

- 基于边缘的执行
- 模块化适配器
- 核心逻辑与特定网站行为的隔离

这种方法允许进行技术探索，而无需将核心系统绑定到特定目标。

## 🧩 系统架构

下图展示了 **Core** 与 **Adapters** 的清晰分离。

```mermaid
flowchart TB
    subgraph Core["核心 (与域无关)"]
        A[请求管道]
        B[适配器接口]
        C[边缘编排]
        D[速率控制与限流]
        E[日志与可观测性]
    end

    subgraph Adapters["适配器 (可选 & 可替换)"]
        F[适配器 网站 A]
        G[适配器 网站 B]
        H[实验适配器]
    end

    A --> B
    B --> F
    B --> G
    B --> H
    C --> A
    D --> A
    E --> A
```

### 核心 (Core)

- 与域无关
- 不包含特定网站的爬取逻辑
- 即使删除所有适配器，也能正常运行

### 适配器 (Adapters)

- 可选模块
- 实现适配器接口
- 可替换、修改或删除而不影响核心
- 用作技术案例示例

## 🔬 适配器流程示例（技术案例研究）

此图展示框架中一个适配器的执行流程。

```mermaid
graph TD
    User[公共用户]

    subgraph Cloudflare_Edge[Cloudflare 边缘]
        Worker[Cloudflare Worker - API 网关]
        KV[(KV 存储)]
    end

    subgraph Residential_Network[住宅网络]
        Client[客户端魔法 - 笔记本 / 家庭服务器]
        Browser[无头浏览器]
    end

    Target[示例目标服务器]

    %% 注册流程
    Client -.->|"1. 自动注册隧道 URL<br/>(启动时)"| Worker
    Worker -.->|"2. 保存隧道 URL"| KV

    %% 请求流程
    User ==>|"3. GET /movies"| Worker
    Worker ==>|"4. 获取活动隧道"| KV
    Worker ==>|"5. 转发请求<br/>(通过安全隧道)"| Client
    Client ==>|"6. 控制"| Browser
    Browser ==>|"7. 执行适配器逻辑"| Target

    %% 返回路径
    Target -.-> Browser
    Browser -.-> Client
    Client -.-> Worker
    Worker -.->|"8. JSON 响应"| User
```

**流程简述：**

1.  客户端适配器在启动时自动向边缘注册隧道。
2.  Cloudflare Worker 将活动端点存储在 KV 中。
3.  公共用户通过边缘访问 API。
4.  Worker 动态选择活动隧道。
5.  请求通过安全隧道转发。
6.  客户端控制无头浏览器。
7.  适配器执行目标特定逻辑。
8.  数据以 JSON 返回。

> 此流程为可选，可删除而不影响核心。
